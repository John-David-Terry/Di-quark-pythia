BREIT FRAME LORENTZ TRANSFORM AUDIT - COMPLETE REPORT
================================================================================

EXECUTIVE SUMMARY
=================

The audit has identified TWO distinct issues:

1. WRONG VECTOR IDENTIFICATION (ROOT CAUSE)
   The definition k_in = k_out - q is NOT giving a physical incoming parton.
   
   Evidence:
   - ISR/FSR OFF: k_in^2 ranges from -6432 to 0 GeV^2 (mean = -213 GeV^2)
   - 89% of ISR/FSR OFF events have |k_in^2| > 1 GeV^2
   - 19% of ISR/FSR OFF events have NEGATIVE k_in energy
   - A physical massless quark should have k_in^2 ≈ 0
   
   This means the k_out read from the event record is NOT the direct
   hard-scattering quark that satisfies k_out = k_in + q with k_in = xP.

2. NONSTANDARD BREIT-LIKE FRAME (SECONDARY ISSUE)
   The LT makes q longitudinal but NOT P longitudinal.
   
   - q^0 ~ 0 ✓ (mean ~4e-5)
   - |qT| ~ 0 ✓ (mean ~3e-5)
   - |P_T| NOT ~0 (mean ~4e-3 GeV)
   
   This is a valid Lorentz transform, but it's not the standard Breit frame.
   The transform aligns q with the z-axis, leaving P with residual transverse
   momentum that fixes the azimuthal direction.

================================================================================
DETAILED FINDINGS
================================================================================

1. LT VALIDITY
--------------
The Lorentz transform IS mathematically correct:
- LT^T g LT = g satisfied to ~1e-14 precision
- det(LT) = 1.000000 exactly
- Norms and dot products are preserved

Conclusion: The transform itself is not broken.

2. BREIT FRAME CONDITIONS
-------------------------
For ISR/FSR OFF (500 events):
  q^0 in Breit: mean = 4.2e-05 GeV (essentially zero)
  |qT| in Breit: mean = 3.4e-05 GeV (essentially zero)
  |P_T| in Breit: mean = 3.9e-03 GeV (NOT zero)

The virtual photon q is correctly transformed to be purely longitudinal.
However, the proton P retains small transverse momentum.

This is a "Breit-like" frame where:
  - q is along -z (correct)
  - P is mostly along +z but with small P_T (nonstandard)
  - The P_T direction is aligned with -x (by construction in M0)

3. k_in IDENTIFICATION - THE ROOT CAUSE
---------------------------------------
The definition used:
  k_in = k_out - q

where k_out is the status=23 quark from the PYTHIA event record.

PROBLEM: k_in defined this way is MASSIVELY OFF-SHELL.

ISR/FSR OFF statistics:
  k_in^2: min=-6432, max=0, mean=-213 GeV^2
  k_out^2: min=0.11, max=23.0, mean=0.41 GeV^2 (nearly on-shell)
  
  Fraction with k_in energy < 0: 18.6%
  Fraction with |k_in^2| > 1 GeV^2: 89.0%

This proves that k_out from the event record is NOT simply k_in + q
where k_in is a nearly-collinear parton with k_in ≈ xP.

Example event (ISR/FSR OFF):
  k_out = (7.46, 4.52, -2.71, 5.27) GeV  → k_out^2 = 0.11 GeV^2 ✓
  q     = (14.51, 4.52, -2.71, -15.74) GeV
  k_in  = k_out - q = (-7.05, 0, 0, 21.01) GeV → k_in^2 = -392 GeV^2 ✗
  
The incoming "parton" has NEGATIVE ENERGY and is deeply spacelike.
This is not a physical parton.

4. WHAT k_out ACTUALLY REPRESENTS
---------------------------------
The find_k_out function in generate_events_raw.py looks for:
  1. First: quarks with status=23 (hard process outgoing)
  2. Then: quarks with status 63-68 (shower-related)
  3. Finally: highest-energy quark

In PYTHIA status conventions:
  - Status 23 = "outgoing hard process partons"

However, even with ISR/FSR OFF, k_out does not satisfy the expected
DIS hard-scattering kinematics. This suggests either:
  a) PYTHIA's status=23 quark includes additional effects
  b) The quark is modified before being written to status 23
  c) There's a mismatch in how PYTHIA handles the DIS vertex

5. STEP-BY-STEP TRANSFORM ANALYSIS
----------------------------------
For the first accepted ISR/FSR OFF event, tracking k_in through the LT:

  LAB:       k_in = (-7.05, 0, 0, 21.01)  |T| = 0.00
  after Mm1: (-34.87, 0, 0, 40.10)        |T| = 0.00
  after M0:  (-34.87, 0, 0, 40.10)        |T| = 0.00
  after M1:  (-47.57, 0, 0, 51.52)        |T| = 0.00
  after M2:  (-47.57, 2.54, 0, 51.46)     |T| = 2.54  ← O(GeV) enters here
  after M3:  (-47.75, 4.89, 0, 51.46)     |T| = 4.89
  after M4:  (20.83, 4.89, 0, 28.32)      |T| = 4.89

The transverse momentum appears at M2 and is amplified by M3.
This is NOT an error in the transform - it's a consequence of
rotating a non-collinear 4-vector to the Breit-like frame.

Key insight: k_in in LAB has |T| = 0, but because k_in is NOT along
the proton direction (it has the wrong E and pz), the rotation that
aligns q with z introduces transverse components.

6. THE REAL PHYSICS
-------------------
For a collinear DIS interaction:
  - Incoming quark: k_in = xP (fraction x of proton momentum)
  - After photon absorption: k_out = k_in + q = xP + q

But the event record shows:
  - k_in = k_out - q has |k_in^2| ~ 100-1000 GeV^2
  - k_in has negative energy in ~20% of events
  - k_in is NOT close to xP even though ISR/FSR is OFF

This means the k_out stored in the event record includes effects
beyond the simple leading-order DIS kinematics.

================================================================================
CONCLUSIONS
================================================================================

1. The Lorentz transform IS VALID and correctly implemented.

2. The frame is BREIT-LIKE (q longitudinal) but NOT STANDARD BREIT
   (P has small transverse momentum).

3. The O(2 GeV) transverse momentum effect is REAL and comes from:
   - k_in = k_out - q is NOT a physical incoming parton
   - The k_out from PYTHIA status=23 is kinematically inconsistent
     with k_in = xP (even with ISR/FSR OFF)
   - When this unphysical k_in is boosted to the Breit frame, it
     acquires large transverse momentum because it's not collinear
     with the proton direction

4. The ROOT CAUSE is the k_in identification, NOT the transform.

================================================================================
RECOMMENDATIONS
================================================================================

1. DO NOT use k_in = k_out - q as the incoming parton momentum.

2. Instead, consider:
   a) k_in = xP (the collinear parton model expectation)
   b) Look up the actual incoming parton from the PYTHIA event history
      (not status=23, but the parton before the hard scattering)
   c) Use the proton direction itself for "jet" axis

3. If using k_in = xP:
   - In LAB: k_in has no transverse momentum
   - In Breit: k_in will have transverse momentum equal to the proton's
     transverse momentum in that frame (small, ~few MeV)

4. Investigate what PYTHIA's status=23 quark actually represents
   in DIS events and whether there's a better way to extract the
   hard-scattering parton momentum.
